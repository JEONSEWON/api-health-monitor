### Reddit r/webdev

**제목:** HTTP 200 OK인데 API가 망가진 경우 어떻게 감지하나요? (Silent Failure)

여러분 팀에서 이런 경험 있으신가요?

```
GET /api/payments/status → 200 OK
응답: {"status": "ok", "payment_id": null}
```

HTTP 200을 반환하지만 payment_id가 null. 사용자는 결제가 됐다고 생각하지만 실제로는 처리 안 됨.

이런 케이스를 탐지하는 방법:

1. **Keyword assertion**: "payment_id" 문자열이 응답에 있는지 확인
2. **Regex**: payment_id가 UUID 형식인지 정규식으로 검사
3. **JSON Path**: `$.payment_id != null` 조건 체크

저는 이걸 자동화하기 위해 checkapi.io를 만들었습니다. 여러분은 어떻게 탐지하고 있나요?

```python
# 직접 구현 예시
import requests, re

def check_silent_failure(url: str) -> bool:
    r = requests.get(url, timeout=10)
    if r.status_code != 200:
        return False
    data = r.json()
    # payment_id가 null이면 실패
    return data.get("payment_id") is not None
```
